# results/graphs/ - Figure Organization

All figures are generated by code in `src/` and saved here automatically via `save_figure()`.

## Naming Convention

```
figure_{VOL}_{NN}.{fmt}
```

- `VOL`: volume prefix - `B` for Volume B labs, `C` for Volume C application
- `NN`: sequential figure number within the volume (01, 02, ...)
- `fmt`: `png` (always) + `svg` (line plots only, not spectrograms)

## How Figures Are Saved

All figures are saved by `src/common/plotting.py:save_figure()`:

```python
save_figure(fig, lab_number=3, fig_id="09")
# -> results/graphs/lab3/figure_B_09.png + .svg

save_figure(fig, lab_number="volume_c/c1", fig_id="01")
# -> results/graphs/volume_c/c1/figure_C_01.png + .svg
```

- `lab_number` as int (1-8): saves to `labN/` with `B` prefix
- `lab_number` as string: saves to that path with volume prefix extracted from the string
- `raster_only=True`: skips SVG (use for spectrograms/heatmaps - dense SVGs crash renderers)

## Regenerating Figures

Each lab/section has a `run_labN(save=True)` or `run_cN(save=True)` function:

```bash
conda activate biosignals
python -c "from src.lab1_dft.lab1 import run_lab1; run_lab1(save=True)"
python -c "from src.volume_c.c1_triage import run_c1; run_c1(save=True)"
```

All figures are deterministic (fixed seed, same data) - regenerating produces identical output.
